Room 2: Object State

Trace the life of a Gadget

You made it into the next part of the project. Now the focus shifts from class blueprints to the state of an object over time.

Study how this Gadget object changes as methods are called. Use your understanding of constructors and mutator methods to discover the next secret word.

The Code Fragment

Consider this class definition (assume it compiles and is already in the project): 

public class Gadget {
    private String name;
    private int charge;

    public Gadget(String gadgetName, int startCharge) {
        name = gadgetName;
        charge = startCharge;
    }

    public void setCharge(int newCharge) {
        charge = newCharge;
    }

    public void boost(int amount) {
        charge += amount;
    }

    public String toString() {
        return name + " (" + charge + "%)";
    }

    public int getCharge() {
        return charge;
    }
}
